Recycled Sound catalogs hearing aids, so Play may surface a Health apps declaration depending on the category you choose. My read: this is a logistics / cataloguing / workflow tool for a donation program, not a medical device, diagnostic, or health-records app — it stores no patient health data. I'd pick category Productivity or Tools (defensible, avoids extra health-compliance forms). Picking Medical is more literally accurate but can trigger additional health declarations. Your call — flagging it rather than silently choosing.

Then: send for review

Once App content shows all green, open Testing → Open testing. Build 56 is already staged there as a draft release — open it, set countries/regions (Australia at minimum), then Save → Review release → Start rollout / Send for review. Google's first review can take a few days; when it clears, the app exits draft and the public opt-in link goes live on that same Open testing page.

What's now one command (uploading future builds)

Build a fresh signed AAB and push it to a track:

cd recycled_sound && flutter build appbundle --release
cd .. && python3 scripts/play_upload.py \
  --aab recycled_sound/build/app/outputs/bundle/release/app-release.aab \
  --track internal --status completed

Tracks: internal | alpha (closed) | beta (open) | production. Use --status draft to stage without releasing. Auth resolves from ~/.claude/keys/recycled-sound-play-publisher.json (override with $RS_PLAY_SA_KEY).

Notes & gotchas
  • versionCode must increase every upload. Driven by pubspec version: x.y.z+N (currently 0.5.0+56). Bump +N before each new upload.
  • Draft-app rule: a brand-new app only accepts status: draft releases on non-internal tracks until it exits draft (first review passes). That's why build 56 is staged on open testing but not yet rolled out.
  • AAB is ~97 MB (bundled TFLite + ML Kit models). Play splits it per-device, so the actual download is much smaller.
  • Google Sign-In would need the upload-key SHA-1 (7D:40:93:A9:…) registered in Firebase — not needed today (anonymous auth on Android).
  • Keystore is resettable under Play App Signing; losing the upload key is recoverable, unlike legacy signing.
  • Default listing language is en-GB — image/listing API calls must target that language code, not en-US.
